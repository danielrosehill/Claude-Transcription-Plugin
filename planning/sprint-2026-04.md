# Sprint — 2026-04

Captured 2026-04-28. Scope and ordering for the next batch of plugin work.

## Cross-cutting directives

These apply to **every** skill in this sprint:

1. **Never overwrite source.** Each transformation writes a new suffixed file. Source audio and prior-stage transcripts are always preserved.
2. **Folder organisation.** When a skill operates on audio, organise outputs:
   - `audio/raw/` — original recordings (untouched)
   - `audio/processed/` — preprocessor outputs (denoised, normalised, silence-trimmed)
   - `transcripts/` — transcript outputs at each refinement stage
   - `context/` — extracted-context outputs
   If working inside an existing project structure, respect it; otherwise create these alongside the source file.
3. **Public repo hygiene.** No secrets committed. Env vars via `${VAR}` interpolation in `.mcp.json`. Add `.gitignore` covering `.env`, `*.local`, audio/transcript scratch dirs.

## Tier 1 — build now

### 1. `preprocess-for-transcription`
Orchestrator skill that prepares audio for AssemblyAI (or any ASR).

Pipeline (in order):
| Pass | Default | Notes |
|---|---|---|
| Denoise | off (flag) | for noisy field recordings |
| Format normalise | always on | mono, 16kHz, opus 24k — single ffmpeg pass |
| Loudness normalise | always on | EBU R128 via ffmpeg `loudnorm` |
| Silence collapse | **always on** | silero-vad, 2.5s threshold, compress to 0.5s, preserve 0.3s head/tail |

Output: `audio/processed/<stem>.preprocessed.opus`. Test against `Job-Search-Planning-0426/context/2026_04_28_16_05_22.opus` to lock silence defaults.

### 2. `iterative-refine`
Stepwise transcript refinement. Never overwrites prior stages.

- **Stage 1** (auto): raw → filler removal + paragraph spacing → `<stem>.cleaned.md`
- **Stage 2** (auto): cleaned → add subheadings → `<stem>.structured.md`
- **Stage 3** (interactive): pause and offer further passes — notes / blog / summary / pdf / translate / stop

Naming: `.raw.md` / `.cleaned.md` / `.structured.md` suffixes. All in `transcripts/`.

### 3. `verify-transcription-env`
Sanity-check skill. Validates `ASSEMBLYAI_API_KEY`, `OPENROUTER_API_KEY`, `GEMINI_API_KEY` are present and reachable via cheap API ping. Reports what works.

### 4. `combine-then-transcribe`
Concatenate multiple audio files into a single job before sending to ASR.

- Accepts list of files (or glob)
- Validates compatibility (or normalises first via preprocessor)
- Concatenates losslessly with ffmpeg concat demuxer
- Optionally inserts short silence between files as boundary marker
- Emits a sidecar `.segments.json` mapping timestamp ranges back to source files (so diarisation/timestamps remain traceable)
- Routes the combined file through `preprocess-for-transcription` then `transcribe-assemblyai`

## Tier 2 — next

### 5. `extract-context`
Transcript → mine personal facts (location, role, projects, preferences) → `context/extracted-{date}.md`. Needs merge-vs-append logic so "Daniel lives in Jerusalem" isn't re-extracted on every memo. Optional write to auto-memory.

### 6. `split-transcript`
Transcript → segment into `prompts/`, `context/`, `action-items/`, `background/` files. Useful for dictated dev briefs that mix multiple intents.

### 7. `translate-transcript`
Source-language transcript → target-language version. Layers on top of cleaning/structuring stages. Gemini or Claude.
